# 程式碼極致精煉器 (Code Extreme Refiner)

## 概述

`code-extreme-refiner` 技能旨在協助 Manus 或其他 AI Agent 將程式碼精煉至最簡潔、高效且無技術債的狀態，同時確保其功能完全正常且保持高品質。此技能的核心目標是透過移除冗餘、扁平化複雜邏輯和應用現代編程模式，來提升程式碼的可讀性、可維護性與執行效率。

## 核心工作流程

當需要對程式碼進行極致精煉時，請遵循以下流程：

### 階段一：識別冗餘與複雜性

在此階段，Agent 應對提供的程式碼進行分析，識別出所有可以被簡化、合併或移除的冗餘部分和複雜結構：

1.  **冗餘判斷**：找出重複的程式碼塊、不必要的變數賦值、多餘的條件判斷（例如 `if (x === true)`）。
2.  **巢狀邏輯**：識別深層的 `if-else` 巢狀結構、多層迴圈等，這些通常會降低可讀性。
3.  **技術債跡象**：尋找過時的語法、不符合現代編程習慣的模式、或可能導致未來維護困難的設計。
4.  **潛在優化點**：評估是否可以利用語言特性（如三元運算子、可選鏈、空值合併運算符）來簡化表達。

**輸出**：一份初步的分析報告，列出所有識別出的冗餘、複雜性問題及潛在的精煉方向。

### 階段二：應用精煉模式與重構

根據第一階段的分析結果，Agent 應應用適當的精煉模式對程式碼進行重構。此階段的重點是：

1.  **提早回傳 (Early Return)**：優先處理最明確、最優先的條件，避免不必要的巢狀結構。
2.  **扁平化邏輯**：將深層的條件判斷或迴圈結構扁平化，提升程式碼的線性流動。
3.  **合併邏輯**：利用布林運算、三元運算子等，將多個簡單的判斷或賦值合併為一行。
4.  **移除冗餘**：刪除所有不必要的程式碼、變數或抽象層。
5.  **保持可讀性**：在精煉的同時，確保程式碼依然清晰、易於理解，避免過度壓縮導致難以閱讀。

**輸出**：重構後的程式碼草稿，以及對每個重構點的簡要說明。

### 階段三：等效驗證與品質確認

重構完成後，必須嚴格驗證精煉後的程式碼與原始程式碼的功能等效性，並確認其品質：

1.  **功能等效性測試**：針對原始程式碼的所有測試案例，執行精煉後的程式碼，確保所有測試都能通過，且輸出結果完全一致。
2.  **邊界條件驗證**：特別針對邊界條件進行測試，確保精煉沒有引入新的錯誤。
3.  **性能評估**：在必要時，對精煉前後的程式碼進行性能評估，確保精煉沒有導致性能下降（理想情況下應有所提升）。
4.  **技術債審查**：再次審查程式碼，確保所有已知的技術債已被清除，且沒有引入新的技術債。
5.  **最終品質確認**：確認程式碼符合「乾淨利落」、「高品質」的標準，包括可讀性、可維護性、簡潔性等。

**輸出**：一份詳細的精煉報告，包含：
*   原始程式碼與精煉後程式碼的對比（例如：diff 格式）
*   功能等效性測試結果
*   性能評估結果（如果適用）
*   最終精煉後的程式碼
*   對未來潛在優化方向的建議
