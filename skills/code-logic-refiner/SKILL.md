# 程式碼邏輯精煉器 (Code Logic Refiner)

## 概述

`code-logic-refiner` 技能旨在透過一套嚴謹的三階段審查流程，協助 Manus 或其他 AI Agent 產出符合高標準的程式碼。此流程不僅確保程式碼功能的正確性，更著重於其邏輯的完善性、避免技術債的產生，並追求程式碼的簡潔與高效。

## 核心工作流程

當需要對程式碼進行深度審查與優化時，請遵循以下三階段流程：

### 階段一：深度邏輯審查

在此階段，Agent 應對提供的程式碼進行首次、全面的邏輯審查。重點關注：

1.  **功能正確性**：程式碼是否完全實現了預期的功能？是否存在邊界條件處理不當的情況？
2.  **潛在 Bug**：是否存在明顯的邏輯錯誤、潛在的運行時錯誤或安全漏洞？
3.  **資源管理**：記憶體、檔案句柄、網路連線等資源是否得到妥善的分配與釋放？
4.  **錯誤處理**：錯誤處理機制是否完善？是否能有效應對異常情況？
5.  **可讀性與一致性**：程式碼風格是否符合規範？變數命名是否清晰？

**輸出**：一份初步的審查報告，列出所有發現的問題、潛在風險及初步的改進建議。

### 階段二：批判性反思與二次驗證

在完成第一階段的審查後，Agent 應對自身的審查結果進行批判性反思，並進行二次驗證。此階段的目標是：

1.  **審查的正確性**：第一階段的審查結論是否客觀、準確？是否存在誤判或遺漏？
2.  **過度設計檢查**：程式碼是否存在過度設計（Over-engineering）的情況？某些複雜的實現是否可以用更簡單的方式達成？
3.  **簡潔性與效率**：在保持功能正確的前提下，程式碼是否可以更簡潔、更高效？是否存在冗餘的程式碼或不必要的複雜性？
4.  **替代方案思考**：針對第一階段提出的改進建議，思考是否存在其他更優的實現方案。

**輸出**：一份反思報告，包含對初步審查結果的修正、補充，以及對不同改進方案的初步評估。

### 階段三：正反辯論與最終重構

此為最終決策階段。Agent 應綜合前兩階段的報告，對所有問題和潛在的改進方案進行正反辯論，權衡利弊，最終產出最優的程式碼重構方案。

1.  **技術債權衡**：評估不同方案對未來維護、擴展可能造成的技術債影響。
2.  **性能與可維護性**：在性能、可維護性、可讀性之間找到最佳平衡點。
3.  **最終決策**：基於充分的論證，選擇最佳的程式碼實現或重構方案。

**輸出**：一份詳細的最終審查報告，包含：
*   原始程式碼的問題總結
*   最終選定的重構方案及其理由
*   重構後的程式碼（或具體的修改建議）
*   對未來潛在優化方向的建議

## 輔助資源

本技能包含以下輔助資源，以協助 Agent 更好地執行審查任務：

### references/

*   `quality-standards.md`：定義了程式碼品質的具體標準，包括無技術債、簡潔性、可維護性等方面的考量。

### templates/

*   `review-report-template.md`：提供結構化的審查報告範本，確保輸出的一致性和完整性。

---

**請注意**：此技能的設計目標是通用於不同的 AI Agent。在執行時，Agent 應根據自身的能力和所處理程式碼的語言特性，靈活運用上述流程和資源。
